# 实施计划

## 任务

将抖音 SPI 验签开关从 `@Value` 轻量注入方式重构为独立配置类，便于后续扩展更多 SPI 配置项。

## 实施步骤

1. 检查项目中 `@ConfigurationProperties` 的常用写法。
2. 新增抖音 SPI 独立配置类，绑定 `plt.order.douyin.spi` 前缀。
3. 修改 `DouyinSpiSignatureService`，使用配置类替代 `@Value`。
4. 保持原配置键和默认值不变，避免影响现有 Nacos 配置。
5. 归档本次实施计划与工作总结。

## 风险与关注点

- 本次只做配置接入方式调整，不改变验签开关语义。
- 若后续需要更细粒度开关，可在同一配置类中继续扩展。
