# createOrder 幂等前置与备注补充实施计划

## 背景
- `createOrder` 当前每次调用都会直接生成新订单号并落库。
- 外部渠道单已经具备 `channelCode + channelOrderNo` 这类天然幂等键。
- 内部单的通用幂等用户希望统一下沉到网关层处理，而不是在核心层强兜底。

## 实施方案
1. 在 `createOrder` 开头增加外部渠道单幂等复用逻辑：
   - `channelCode` 和 `channelOrderNo` 均非空时，按这两个字段查询已存在订单；
   - 命中时直接返回既有 `orderNo`。
2. 在方法内增加注释，明确：
   - 外部单在核心层做幂等复用；
   - 内部单通用幂等暂由网关统一处理。
3. 在落单前补充 TODO 注释，说明库存域接入前此处仍不满足真实上线顺序要求。

## 边界说明
- 本次不引入方法级分布式锁，避免内部空 `channelOrderNo` 场景被错误串行化。
- 本次不实现库存预占或价格校验。
