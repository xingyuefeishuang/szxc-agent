# 实施计划

## 任务

为抖音 SPI 验签增加可通过 Nacos 动态开启/关闭的配置开关。

## 实施步骤

1. 检查订单模块现有动态配置接入方式。
2. 在 `DouyinSpiSignatureService` 中新增验签开关配置项。
3. 通过 `@RefreshScope` + `@Value` 使开关支持 Nacos 动态刷新。
4. 在开关关闭时直接跳过验签并记录日志，在开启时保持原有验签逻辑不变。
5. 归档本次实施计划与工作总结。

## 风险与关注点

- 关闭验签后，抖音 SPI 接口将不再校验请求签名，仅适合联调或应急场景。
- 本次未新增独立配置类，采用轻量接入方式以减少改动面。
