# 外部接口 DTO 注释规则沉淀计划

## 背景

抖音 SPI 联调过程中，单纯补字段无法解决后续维护成本问题。  
如果 DTO 中缺少协议分组、字段语义说明和 TODO 保留约束，后续开发仍需要反复翻文档，且容易把未完成事项误删。

## 目标

1. 将“外部接口 DTO 注释与分组规范”沉淀为仓库级规则。
2. 规则不绑定抖音场景，适用于所有第三方 SPI / OpenAPI / 回调协议 DTO。
3. 明确要求保留已有协议说明、业务注释和 TODO。

## 实施步骤

1. 在 `.agent/rules/AI_BEHAVIOR_RULES.md` 中追加规则。
2. 规则内容覆盖：
   - 按协议场景或版本分组字段
   - 字段和嵌套结构补充中文注释
   - 标明关键协议约束
   - 不得擅自删除原有 TODO 和协议说明
3. 归档本次规则沉淀的实施计划与总结。
