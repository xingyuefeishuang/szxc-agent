# NONE 履约类型状态语义补充计划

## 背景
- 已新增 `fulfillment_type = NONE`，表示支付成功后不立即履约。
- 当前订单状态机没有“待人工履约/待发货”之类的新状态。

## 约定
1. 在现有状态机不新增状态的前提下，`NONE` 类型订单支付成功后保持在 `PAID`。
2. `VOUCHER` 类型订单仍按现有语义推进到 `DELIVERING`。
3. 后续若新增更细状态，再由 `NONE` 对应策略调整状态推进。

## 本次落地
1. 在 `NoopOrderFulfillmentStrategy` 注释中明确保持 `PAID` 的语义。
2. 在履约类型设计文档中补充当前状态语义说明。
