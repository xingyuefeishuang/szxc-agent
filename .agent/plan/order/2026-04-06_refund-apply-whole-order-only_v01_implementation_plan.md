# 退款申请整单收敛实施计划

## 背景
- 当前 `RefundAddDO` 暴露了 `orderItemId`，但 `applyRefund(...)` 实现仍然完全按整单处理。
- 这会让接口契约看起来支持部分退款，实际却冻结整单凭证并推进整单状态。

## 实施方案
1. 将退款申请接口文案明确为“当前仅支持整单退款”。
2. 在 `RefundAddDO` 上补充整单退款相关注释与必要参数校验。
3. 在 `RefundApplyServiceImpl.applyRefund(...)` 中显式拒绝传入 `orderItemId` 的请求。
4. 保留 `orderItemId` 字段作为未来扩展位，但当前版本不启用。

## 边界说明
- 本次不实现真正的部分退款。
- 本次不改审核通过/驳回的状态语义，仅先把申请对象边界收紧。
