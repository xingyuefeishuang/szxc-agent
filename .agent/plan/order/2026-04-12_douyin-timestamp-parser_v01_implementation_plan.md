# 实施计划

## 任务

修复抖音景区团购发码场景下的时间戳解析错误，避免异常时间值写入 `o_fulfillment_voucher.valid_start_time` 与 `valid_end_time`。

## 实施范围

- 抽出抖音时间戳统一解析能力，兼容秒、毫秒、微秒、纳秒
- 将 `ScenicGroupbuyDouyinSolution` 的 B11 发码时间转换接入统一解析
- 为时间戳解析新增单元测试
- 归档本次任务实施计划与总结

## 实施步骤

1. 复盘报错链路，确认 `start_time/expire_time` 来源与当前转换逻辑。
2. 新增统一时间戳解析工具，按数量级归一化到毫秒并做合理区间校验。
3. 替换 B11 发码逻辑中的旧 `toDate` 转换，并补充异常日志上下文。
4. 新增单元测试覆盖四种精度与异常值。
5. 尝试执行模块级测试/编译验证并记录阻塞信息。

## 风险与约束

- 抖音文档在不同接口中对时间戳精度定义不一致，必须兼容处理。
- 本地 Maven 校验依赖仓库固定 JDK 环境；若环境不满足 JDK 17，将无法完成编译验证。
- 本次仅修复时间解析与接入点，不扩展数据库结构和业务状态机逻辑。
