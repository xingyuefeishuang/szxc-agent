# 发券回调接口

> **收藏**: [我的收藏](#)

## 接口说明

本接口用于服务商**异步返回发码结果**时通知抖音开放平台（以下简称抖音）最终状态。

*   **触发场景**：当抖音通过“发放凭证（SPI）”接口向服务商发起发码申请后，若服务商无法立即同步返回结果（需异步处理发码），必须通过此接口异步通知最终结果。
*   **超时机制**：若抖音在 **10 分钟** 内未收到通知，将自动发起退款。
*   **同步场景**：若能同步完成发码，则**无需**调用该接口。
*   **重试机制**：服务商调用该接口得到的响应中 `error_code` 若为 `0` 则表示通知成功，非 `0` 则代表失败，需持续重试直至成功。
*   **失败处理**：发码失败时会额外返回 `result=2`。
*   **补发凭证**：如需补发凭证码，可在抖音发起退款时通过构造接口响应拒绝退款和补发新码。

### 前提条件
调用接口前，需参考《景区行业解决方案》完成“接入前期准备”工作。

### 使用限制
*   **SLA**: 支持的最大 QPS: **4000**。

## 请求参数

### 请求头 (Headers)

| Key | 字段类型 | 示例 | 描述 |
| :--- | :--- | :--- | :--- |
| `access-token` | String (必填) | `clt.943da...` | 调用 `https://open.douyin.com/oauth/client_token/` 生成的 token |
| `content-type` | String (必填) | `application/json` | 固定值 |
| `Rpc-Transit-Life-Account` | String | - | 来客商户根账户 ID |

### 请求体 (Body)

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `order_id` | String (必填) | 抖音侧的订单号（抖音通过调用发放凭证 SPI 时给出） |
| `result` | Int64 | 发券结果：`1`: 成功`2`: 失败 |
| `third_order_id` | String | 如发券成功，将第三方订单号回传抖音（由开发者系统生成） |
| `fail_reason` | String | 失败原因（参考文档下的“发券失败原因”枚举） |
| `fail_reason_desc` | String | 失败原因描述 |
| `additional_info_list` | List | 凭证附加信息（**景区解决方案不适用**） |
| `certificates` | List | 券列表，批量发码使用（**景区解决方案不适用**） |
| `voucher` | Struct | **景区预售券凭证**（**仅景区行业使用该字段**）由开发者系统生成。结构见下方详解。 |
| `vouchers` | List | **仅「日历票一单多份套票」场景使用该字段**。由开发者系统生成。结构同 `voucher` 字段。 |

#### `voucher` / `vouchers` 对象结构详解

该结构包含入园项目和自定义项目信息。

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `entrance` | Struct | **景区的入园项目** |
| `projects` | List | **可自定义的项目**（例：景区项目索道 A、索道 B 等） |

##### `entrance` (入园项目) 字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `project_id` | String (必填) | 项目唯一标识，不能重复（核销时需要） |
| `certificate_nos` | Array | 券号凭证，在 C 端展示为数字串，最多限 100 个 |
| `credentials` | List | 身份凭证列表，最多限 100 个 |
| `gmcode_imgs` | Array | - |
| `id_cards` | Array | 身份证号码，最多限 100 个 |
| `qrcodes` | Array | 二维码凭证，可以是一串数字，也可以是一个 URL，最多限 100 个 |
| `urls` | Array | url 电子凭证，最多限 100 个。每个 URL 长度不能超过 512 |

###### `credentials` (身份凭证) 列表项字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `credential_no` | String | 证件号 |
| `credential_type` | Int64 | 证件类型：`1`: 身份证`2`: 港澳通行证`3`: 台湾通行证`4`: 回乡证`5`: 台胞证`6`: 护照 |

##### `projects` (自定义项目) 列表项字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `name` | String (必填) | 自定义的项目名称（例：景区 XXXX 索道） |
| `project_id` | String (必填) | 项目唯一标识，不能重复（核销时需要） |
| `certificate_nos` | Array | 券号凭证，在 C 端展示为数字串，最多限 100 个 |
| `credentials` | List | 身份凭证列表，最多限 100 个 (结构同上) |
| `gmcode_imgs` | Array | - |
| `id_cards` | Array | 身份证号码，最多限 100 个 |
| `qrcodes` | Array | 二维码凭证，最多限 100 个 |
| `urls` | Array | url 电子凭证，最多限 100 个 (单条长度<=512) |

## 请求示例 (Java)

```java
import com.aliyun.tea.TeaException;
import com.douyin.openapi.client.Client;
import com.douyin.openapi.client.models.*;
import com.douyin.openapi.credential.models.Config;

public class Main {
    public static void main(String[] args) {
        try {
            // 配置 ClientKey 和 ClientSecret
            Config config = new Config().setClientKey("tt******").setClientSecret("cbs***"); 
            Client client = new Client(config);
            
            /* 
               构建请求参数：
               1. token: 可自行维护或调用 SDK 获取 (注意互刷问题)
               2. header: SDK 默认填充 content-type，其他请求头按需添加
            */
            CreateCallbackRequest sdkRequest = new CreateCallbackRequest();
            sdkRequest.setAccessToken("k23cd*****");
            sdkRequest.setFailReason("O8XnZvLTIG");       // 失败原因
            sdkRequest.setFailReasonDesc("4Pp6bABcmK");   // 失败原因描述
            sdkRequest.setOrderId("Wfp9CXYqDy");          // 抖音订单号
            sdkRequest.setResult(1);                      // 1=成功, 2=失败
            sdkRequest.setThirdOrderId("mr5yzF12KX");     // 第三方订单号
            
            // 发送请求
            CreateCallbackResponse sdkResponse = client.CreateCallback(sdkRequest);
            
        } catch (TeaException e) {
            System.out.println("TeaException: " + e.getMessage());
        } catch (Exception e) {
            System.out.println("Exception: " + e.getMessage());
        }
    }
}
```

## 响应参数

### 响应体 (Body)

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `extra` | Struct (必填) | 扩展信息 |
| `data` | Struct | 业务数据 |
| `certificate_info_list` | List | 券关系列表（只有请求中回传了第三方订单号时，会返回） |

#### `extra` 对象字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `description` | String (必填) | 描述 |
| `error_code` | Int32 (必填) | 错误码 (`0` 表示成功) |
| `logid` | String (必填) | 抖音日志 id，可用于排查问题 |
| `now` | Int64 (必填) | 调用时间，单位为秒 (s) |
| `sub_description` | String | 子描述 |
| `sub_error_code` | Int32 | 子错误码 |

#### `data` 对象字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `description` | String (必填) | 错误描述 |
| `error_code` | Int32 (必填) | 错误码 |
| `certificate_info_list` | List | 券关系列表（同 `extra` 中的列表） |

#### `certificate_info_list` 列表项字段

| Key | 字段类型 | 描述 |
| :--- | :--- | :--- |
| `certificate_id` | String (必填) | 券 ID（抖音侧的券唯一标识） |
| `code` | String (必填) | 三方码 |
