# 景区团购发码回调接口 V2

**更新时间**: 2025-07-31 21:42:04  
**接口场景**: 服务商异步通知抖音侧发码结果（支持多 SKU）

---

## 1. 接口说明

当服务商接入**异步发码**流程，或在同步发码超时后完成发码时，需调用此接口将最终的凭证码（或失败状态）回传给抖音。

*   **触发时机**: 服务商内部完成出码逻辑后。
*   **核心作用**: 更新抖音侧订单的凭证状态。只有回调成功后，用户才能在抖音端看到凭证并入园。
*   **支持能力**: 支持单次回调多个 SKU 的凭证信息。
*   **权限要求**: `life.capacity.scenic_promissory_callback_code`

### 基本信息

| 项目 | 描述 |
| :--- | :--- |
| **HTTP URL** | `https://open.douyin.com/goodlife/v1/trip/scenic/groupon/certificate/callback/` |
| **HTTP Method** | `POST` |
| **Scope** | `life.capacity.scenic_promissory_callback_code` |
| **Content-Type** | `application/json` |

## 2. 请求参数

### 请求头 (Headers)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `access-token` | String | 是 | 调用 `client_token` 接口生成的凭证。 |
| `content-type` | String | 是 | 固定值 `"application/json"`。 |
| `Rpc-Transit-Life-Account` | String | **是** | **来客商户根账户 ID** (必填)。 |

### 请求体 (Body)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `order_id` | String | 是 | 抖音侧订单号（与发码申请时的 `order_id` 一致）。 |
| `result` | Enum | 是 | 发码结果状态：`0`: **发码中** (Doing)`1`: **发码成功** (Success)`2`: **发码失败** (Fail) |
| `certificate_info_list` | List | 否 | 三方码列表。**当 `result=1` 时必填**。结构见下方 [凭证列表结构](#凭证列表结构)。 |

#### 凭证列表结构 (`certificate_info_list` Item)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `certificate_id` | String | 是 | **凭证 ID** (对应发码请求中的 `certificate_id`)。*(注：文档表格中标注为 Int64，但示例及常规逻辑通常为 String，请以实际对接为准，建议按 String 处理)* |
| `project_list` | List | 是 | 项目凭证信息列表。结构见下方 [项目列表结构](#项目列表结构)。 |

#### 项目列表结构 (`project_list` Item)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `project_id` | String | 是 | 项目 ID。 |
| `name` | String | 是 | 项目名称。 |
| `certificate_list` | List | 否 | 凭证码列表。若仅凭证件入园，可为空。结构见下方 [凭证码结构](#凭证码结构)。 |
| `credential_list` | List | 否 | 证件信息列表。结构见下方 [证件结构](#证件结构)。 |

#### 凭证码结构 (`certificate_list` Item)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `certificate_no` | String | 是 | 凭证号/码/URL。 |
| `ccertificate_type` | Enum | 是 | 凭证类型：`1`: **QrCode** (二维码内容)`2`: **Url** (链接)`3`: **Code** (纯代码) |

#### 证件结构 (`credential_list` Item)

| Key | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `credential_no` | String | 否 | 证件号。 |
| `credential_type` | Enum | 否 | 证件类型：`1`: **ID_CARD** (身份证)`2`: **HM_PERMIT** (港澳通行证)`3`: **TW_PERMIT** (台湾通行证)`4`: **REENTRY_PERMIT** (回乡证)`5`: **TW_MAINLAND_TRAVEL_PERMIT** (台胞证)`6`: **PASSPORT** (护照) |

## 3. 请求示例

### Java SDK 示例

```java
import com.aliyun.tea.TeaException;
import com.douyin.openapi.client.Client;
import com.douyin.openapi.client.models.CertificateCallbackRequest;
import com.douyin.openapi.client.models.CertificateCallbackResponse;
import com.douyin.openapi.credential.models.Config;

public class Main {
    public static void main(String[] args) {
        try {
            // 配置 ClientKey 和 Secret
            Config config = new Config()
                .setClientKey("tt******")
                .setClientSecret("cbs***"); 
            
            Client client = new Client(config);

            // 构建请求参数
            CertificateCallbackRequest sdkRequest = new CertificateCallbackRequest();
            
            // 1. 设置 AccessToken (可自行维护或调用 SDK 获取)
            sdkRequest.setAccessToken("k23cd*****");
            
            // 2. 设置订单号
            sdkRequest.setOrderId("PXNJbyfZDK");
            
            // 3. 设置结果状态 (0:发码中，1:成功，2:失败)
            sdkRequest.setResult(1); 
            
            // 4. 设置凭证列表 (result=1 时必填，此处省略具体 List 构建代码，参考上方参数结构)
            // sdkRequest.setCertificateInfoList(...);

            // 发起回调
            CertificateCallbackResponse sdkResponse = client.CertificateCallback(sdkRequest);
            
            System.out.println("Callback Success: " + sdkResponse.getBody());
            
        } catch (TeaException e) {
            System.out.println("TeaException: " + e.getMessage());
        } catch (Exception e) {
            System.out.println("Exception: " + e.getMessage());
        }
    }
}
```
