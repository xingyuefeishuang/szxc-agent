**更新时间：** 2026-03-09

**无需业务授权**

---

#### `meituan.trip.order.refund.notice` (订单退款通知)

##### 使用场景

美团发起合作方退款申请且商家审核退款通过后，可使用该接口通知美团退款审核完成，美团将为用户进行后续退款操作

##### 接口说明

1.  合作方通过该接口返回退款审批结果时，退款流水号为退款唯一标识
2.  如果此退款流水号已经退款成功, 请返回退款成功（code 200）或已退款（code 601）
3.  退款其他状态请根据该页的“返回码解释”返回对应信息

#### 请求参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| charset | String | 是 | 字符编码 |
| developerId | Long | 是 | 服务商身份标识 |
| sign | String | 是 | 数字签名 |
| businessId | int | 是 | 业务类型 id，固定传 66 |
| version | String | 是 | 版本号，固定传 2 |
| timestamp | Long | 是 | 时间戳 |
| biz | jsonString | 是 | 业务请求参数 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | int | 是 | 交互码，必须回传；200 为退款成功，601 为已退款，602 为审批中，604 为产品本身不能退款，605 为产品不能线上退款，606 为申请张数超过可退款张数，607 为产品不在可退款时间内，608 为退款金额校验失败，699 为其它退款错误 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | 是 | 合作方 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | 是 | 交互描述，如错误信息等 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | object<Body> | 是 | 层级及价格库存信息数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderId | long | 是 | 美团订单 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | String | 是 | 合作方订单 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ refundId | String | 是 | 美团退款流水号，退款申请唯一标识，与美团退款请求 `refundId` 对应 |

```json
    "developerId": "123456",
    "businessId": "66",
    "sign": "6c9eedc09b5b33245bbcf0c9b24cb8c1f274test",
    "version": "2",
    "timestamp": "1772269850"
}
```

#### 响应参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | String | 是 | 响应状态码，code 为 OP_SUCCESS 表示响应成功，其他状态码表示响应失败。具体系统错误码可以参考系统错误码。 |
| msg | String | 是 | 错误消息 |
| traceId | String | 是 | 请求消息唯一标识。 |
| data | Object | 是 | 业务数据，为 Json 对象，如果请求错误，该字段可能会展示错误的详细信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | int | | 交互码，200 为成功，300 为失败 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | | 描述信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | | 合作方 ID |
