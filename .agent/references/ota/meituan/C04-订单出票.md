### 订单出票

**更新时间：** 2026-03-19

**无需业务授权**

---

#### 消息场景说明

`meituan.trip.order.pay` (订单出票)

##### 使用场景

美团调用此接口到合作方支付订单

##### 接口说明

1.  如果合作方返回“出票失败”（`code`非 200）, 美团会立刻给用户退款。
2.  如果合作方返回“出票中”不返回结果或出现接口异常（如超时），美团多次会调用查询接口进行查询，根据查询接口返回的订单状态进行后续处理；若美团多次查询后依然无结果，美团认为合作方出票状态未知，进入人工处理环节。
3.  该接口返回 `code` 200，代表着“用户可以入园”。建议如果 30 秒内确定能出票商家还是同步回传出票结果，其余的可以考虑 598 异步出票。
4.  如果你是要向上游二次直连出票，请务必返回“598 出票中”&通过出票通知接口返回最终的出票结果。要求 15 分钟内回传异步出票结果。
5.  该接口通用超时底线时间：连接 30s 读取 30s，可调整但不能总和不能超过 1min。
6.  出票失败的 `describe` 要返回了真实的失败原因，不能用“其他”代替。`code` 按照美团给出的对接。
7.  如果 `code` 200，且是要回传凭证的产品，`voucherType`、`voucher` 都一定有且正确。
8.  需要码核销 `voucherType=1` 订单买多张就要有多个码。不需要码核销 `voucherType=0` 的订单最多只能有一个码

#### 公共参数

*   **HTTP URL：** `https://baseUrl`（供应商提供）（仅支持 https 协议且端口为 443）
*   **HTTP Param：** param 遵循 JSON 格式，ContentType 为 `application/x-www-form-urlencoded`, charset=utf-8
*   **HTTP 方法：** POST
*   **消息推送公共参数：** 消息推送公共参数
*   **消息响应公共规范：** 消息响应公共规范

#### 供应商返回参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | Integer | 是 | 0 为正常，非 0 为异常 |
| message | String | 否 | 请求失败原因说明 |
| data | String | 否 | 扩展数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | Integer (int32) | 是 | 交互码，必须回传；200 为出票接口成功，501 为虚拟账户余额不足，502 为购买规则校验不通过，503 为库存不足，504 为接口超时，505 为产品错误，506 为接口错误，507 为预付款金额不足，599 为其他，598 为出票中 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | 是 | 交互描述，如错误信息等。失败时要返回真实的失败原因，不能用“其他”代替，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | Integer (int32) | 是 | 合作方 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | object<MtpOrderPayResponseBody> | 否 | 支付订单返回内容对象, 出票成功时必传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderId | Integer (int64) | 是 | 美团订单 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | String | 是 | 合作方订单 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherType | Integer (int32) | 是 | 凭证码类型：0 为一码多验或不核验凭证码，1 为一码一验。 现 `voucherType` 只有 0 和 1，`voucherType=1` 对应一码一验，凭证码数量必须与订单里的票数对应，核销时按照回传的凭证码进行核销；其他情况需回传 `voucherType=0`，此种情况下退款请求不会传凭证码给商家，核销时商家也可以不回传凭证码（按 `usedQuantity` 进行核销），需要码入园的产品商家照常返回凭证码即可，凭证码会照常展示。 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ vouchers | array<String> | 是 | 出票成功时为必传字段；凭证码数组，每个凭证码的最大长度为 varchar(20)；凭证码不区分大小写，不允许包含重复的凭证码，否则会导致出票失败。 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherPics | array<String> | 否 | 图片凭证码数组，出票成功时为必传字段；传图片链接地址, 有多个时, 顺序必须与 `vouchers` 的顺序一致；每个地址指向一张可下载的图片 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherAdditionalList | array<VoucherAddition> | 否 | 凭证附加信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucher | String | 是 | 必须存在于 `MtpOrderPayResponseBody` 中的 `vouchers` 中 `voucher` |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ additionMap | Map<Integer,String> | 是 | 凭证附加信息，key 值为“1”，表示为手机尾号后四位，value 为手机尾号后四位的值 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ realNameType | Integer(int32) | 否 | 实名制订单标识，取值：0、1； 0 代表非实名制订单(退款和核销无需传身份证信息)；1 代表实名制且需要凭证码订单(退款和核销需要凭码+身份证信息，即需要券码和游客信息相匹配)；不传默认为 0 非实名制订单 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialList | array<Credential> | 否 | 实名制证件、凭证码信息，`realNameType` 字段为 1 时必传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialType | Integer(int32) | 是 | 证件类型，0 表示身份证，1 表示护照，2 表示军官证，3 表示台胞证，4 表示港澳通行证，5 表示大陆居民往来台湾通行证，6 表示港澳居民来往内地通行证，7 表示外国人永久居留证，8 表示港澳台居民居住证，9 表示军人证 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialNo | String | 是 | 证件号 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucher | String | 是 | 该证件号对应的凭证码 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionalList | array<OrderAddition> | 否 | 订单附加信息列表 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionKey | Integer(int32) | 是 | 订单附加信息 Key，1 代表座位号，10 代表入园凭证 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionValue | String | 是 | 订单附加信息 Value，如果 `orderAdditionKey` = 1，则代表座位号的具体信息。如果有多个座位号，需要全部放到一起。`OrderAdditionValue` 字段推荐格式为 用户信息：座位号（姓名&凭证码&脱敏证件号&手机尾号：座位号信息），换行符使用 `\r\n`。 示例：游客:张三 座位号：B2-7；\r\n游客:李四 座位号：B2-8；\r\n游客:王二 座位号：B2-9；\r\n 如果 `orderAdditionKey` = 10，则代表入园凭证信息，入园凭证可传多个，其中 `type` = 1 表示图片，`type` = 2 表示 PDF 文件 示例：`"[{\"type\":1,\"value\":\"http://image.partner.com/pic/voucher001.png\"},{\"type\":2,\"value\":\"http://pdf.partner.com/pdf/voucher002.pdf\"}]"` |

| 503 | 库存不足 |  |
| 505 | 产品错误 |  |
| 507 | 预付款金额不足 |  |
| 599 | 其他 |  |
| 598 | 出票中 |  |
