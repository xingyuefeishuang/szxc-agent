### 订单查询

**更新时间：** 2026-03-19

**无需业务授权**

---

#### 消息场景说明

`meituan.trip.order.query` (订单查询)

##### 使用场景

1.  美团主动查询合作方订单状况，包括订单状况与凭证码使用状况。
2.  订单出票接口异常时（如超时），美团多次会调用本接口查询订单状态，进行后续处理，所以一定要返回最最最正确的订单状态 `orderstatus`、订单上退款/使用的数量、订单 `voucher` 的 `status`。
3.  对于合作方无法推送核销的情况，美团可以通过该查询接口主动查询订单与凭证码的状态进行核销。
4.  `orderstatus=4` 代表着“用户可以入园”，如果你要向上游二次直连出票，请务必 `orderstatus=5`。（请看下方的响应事例）。

---

#### 公共参数

*   **HTTP URL：** `https://baseUrl`（供应商提供）（仅支持 https 协议且端口为 443）
*   **HTTP Param：** param 遵循 JSON 格式，ContentType 为 `application/x-www-form-urlencoded`, charset=utf-8
*   **HTTP 方法：** POST
*   **消息推送公共参数：** 消息推送公共参数
*   **消息响应公共规范：** 消息响应公共规范

#### 美团请求参数

| 参数 | 类型 | 是否必传 | 描述 |
| :--- | :--- | :--- | :--- |
| msgType | Integer | 是 | 接口类型 ID：6610029 |
| developerId | Integer | 是 | 开发者 ID，在美团开发者网页直连申请时生成 |
| opBizCode | String | 否 | 业务实体 id，参数与开发者 ID 一致 |
| businessId | Integer | 是 | 产线 ID，默认传 66 |
| sign | String | 是 | 消息的安全签名，可以根据此签名判断消息真伪。开发者需要对请求消息的签名进行验证，签名规则见：签名规则 |
| msgId | String | 是 | 本条请求的唯一标识，可用于去重判断 |
| timestamp | Long | 是 | 消息推送时间戳（秒） |
| message | String | 是 | json 格式的消息数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | 是 | 合作方 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | object<MtpOrderQueryRequestBody> | 是 | 查询订单请求内容对象 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderId | long | 是 | 美团订单 Id |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | string | 否 | 合作方订单 ID |

#### 请求示例

> **注意：** 回调消息统一通过 HTTPS 请求推送，Post 方法，ContentType 为 `application/x-www-form-urlencoded`。开发者需要按照规范实现 HTTPS 接口，以接收推送的消息。

**请求示例**
```json
{
    "msgType": "6610029",
    "opBizCode": "1711591987",
    "developerId": "123456",
    "businessId": "66",
    "sign": "4656285a4c2493e279d929b8b9f4e29310datest",
    "msgId": "3702923312382104528",
    "timestamp": "1711592316026",
    "message": "{\"partnerId\":1234,\"body\":{\"orderId\":1234567,\"partnerOrderId\":\"partner-order-000001\"}}"
}
```

| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherType | Integer (int32) | 是 | 凭证码类型：0 为不需要码核销，1 为需要码核销。 voucher type 原语义为：0 为不回传凭证，1 为一码一验，2 为一码多验，原语义已废弃。现 voucher type 只有 0 和 1，voucher type=1 对应之前的一码一验，凭证码数量必须与订单里的票数对应，核销时按照回传的凭证码进行核销；其他情况需回传 voucher type=0，此种情况下退款请求不会传凭证码给商家，核销时商家也可以不回传凭证码（按 used quantity 进行核销），需要码入园的产品商家照常返回凭证码即可，凭证码会照常展示。 |

| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ quantity | Integer (int32) | 是 | 与 status 配合使用，此状态下的凭证代表的数量 |

| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherAdditionalList | array<VoucherAddition> | 否 | 凭证附加信息 |

| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ additionMap | Map<Integer,String> | 是 | 凭证附加信息，key 值为“1”，表示为手机尾号后四位，value 为手机尾号后四位的值 |

| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ status | Integer (int32) | 是 | 证件状态，0 为未使用，1 为已使用，2 为已退款 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionalList | array<OrderAddition> | 否 | 订单附加信息列表 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionKey | Integer (int32) | 是 | 订单附加信息 Key，1 代表座位号，10 代表入园凭证 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionValue | String | 是 | 订单附加信息 Value，如果 `orderAdditionKey` = 1，则代表座位号的具体信息。如果有多个座位号，需要全部放到一起。`OrderAdditionValue` 字段推荐格式为 用户信息：座位号（姓名&凭证码&脱敏证件号&手机尾号：座位号信息），换行符使用 `\r\n`。 示例：游客:张三 座位号：B2-7；\r\n游客:李四 座位号：B2-8；\r\n游客:王二 座位号：B2-9；\r\n 如果 `orderAdditionKey` = 10，则代表入园凭证信息，入园凭证可传多个，其中 `type` = 1 表示图片，`type` = 2 表示 PDF 文件 示例：`"[{\"type\":1,\"value\":\"http://image.partner.com/pic/voucher001.png\"},{\"type\":2,\"value\":\"http://pdf.partner.com/pdf/voucher002.pdf\"}]"` |
