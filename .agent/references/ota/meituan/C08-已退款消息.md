### 已退款消息

**更新时间：** 2026-03-19

**无需业务授权**

---

#### `meituan.trip.order.refunded.info` (已退款消息)

1.  美团对如下三种退款不会走商家退款审核：美团客服和商家核实可以退款的客服操作退款、商家审核退款申请超过三个工作日美团自动退款、商家支付结果未知美团多次通过订单查询也没查询到支付成功时退款。
2.  对于上述三种退款，美团会通过该接口告知商家系统商，也会通过邮件形式告知商家人员。请系统商及时撤销支付凭证避免用户既得到了退款又能入园，因此建议商家对接该接口。
3.  支付结果商家明确返回“支付失败”的这种情况，不会通过该接口告知商家。
4.  三种退款通知 `type` 对应在“已退款消息类型”映射表中。

---

#### 公共参数

*   **HTTP URL：** `https://baseUrl`（供应商提供）（仅支持 https 协议且端口为 443）
*   **HTTP Param：** param 遵循 JSON 格式，ContentType 为 `application/x-www-form-urlencoded`, charset=utf-8
*   **HTTP 方法：** POST
*   **消息推送公共参数：** 消息推送公共参数
*   **消息响应公共规范：** 消息响应公共规范

---

#### 请求示例

> **注意：** 回调消息统一通过 HTTPS 请求推送，Post 方法，ContentType 为 `application/x-www-form-urlencoded`。开发者需要按照规范实现 HTTPS 接口，以接收推送的消息。

**非实名制订单已退款消息请求**
```plain text
{
    "msgType": "6610033",
    "opBizCode": "1711591987",
    "developerId": "123456",
    "businessId": "66",
    "sign": "4656285a4c2493e279d929b8b9f4e29310da8b3b",
    "msgId": "3702923312382104528",
    "timestamp": "1711592316026",
    "message": "{\"partnerId\":666,\"body\":{\"orderId\":1234567,\"partnerOrderId\":\"partner-order-000001\",\"refundSerialNo\":\"1234567-20180101160044458\",\"voucherList\":[\"voucher_001\"],\"voucherType\":2,\"refundPrice\":10000,\"refundMessageType\":1,\"operator\":\"客服李四\",\"reason\":\"客服记录：致电商家电话xxxxxxx 赵先生： 可以退款100元 客服同意退款\",\"refundTime\":\"2018-01-01 12:05:00\",\"count\":1}}"
}
```

#### 供应商返回参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | Integer | 是 | 0 为正常，非 0 为异常 |
| message | String | 否 | 请求失败原因说明 |
| data | String | 否 | 扩展数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | Integer (int32) | 是 | 交互码，200 为成功，300 为失败, 必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | 是 | 交互描述，如错误信息等，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | Integer (int32) | 是 | 合作方 ID，必须回传 |

---

#### 返回示例

```json
{
    "code": 0,
    "message": "success",
    "data": "{\"code\":200,\"describe\":\"退款\",\"partnerId\":666}"
}
```
