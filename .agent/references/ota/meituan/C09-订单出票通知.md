### 订单出票通知

**更新时间：** 2026-03-09

**无需业务授权**

---

#### `meituan.trip.order.pay.notice` (订单出票通知)

第三方通过本接口异步向美团返回出票结果。
异步通知前，请在出票接口返回（code：598）将订单置为支付中，然后调用本接口通知出票结果。

合作方通过此接口回传订单出票结果。美团根据出票结果，将订单置为「支付成功」或「支付失败」。

如果出票接口未收到订单支付结果，导致订单变成「支付失败」，合作商再通过本接口通知出票结果无效。

每个订单只能通知一种出票结果(即不允许通知出票成功后, 又通知出票失败; 反之亦然)，请商家在确认出票最终结果后再通知美团。 通知有效时间为美团调用出票接口后的 10 分钟以内。

美团收到出票通知后，会返回确认成功（code：200）或失败（code：300), 当收到失败结果时, 请合作方及时撤销支付凭证避免用户入园。

#### 请求参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| charset | String | 是 | 字符编码 |
| developerId | Long | 是 | 服务商身份标识 |
| sign | String | 是 | 数字签名 |
| businessId | int | 是 | 业务类型 id，固定传 66 |
| version | String | 是 | 版本号，固定传 2 |
| timestamp | Long | 是 | 时间戳 |
| biz | jsonString | 是 | 业务请求参数 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ issueType | int | 是 | 合作方出票类型, 1: 出票成功, 2: 出票失败 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | string | 是 | 交互描述，如错误信息等，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | 是 | 第三方的 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | object<MtpOrderPayResponseBody> | 是 | 订单消费通知内容 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderId | long | 是 | 美团订单 id |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | string | 是 | 合作方订单 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherType | int | 是 | 凭证码类型：0 为不需要码核销，1 为需要码核销。 voucher type 原语义为：0 为不回传凭证，1 为一码一验，2 为一码多验，原语义已废弃。现 voucher type 只有 0 和 1，voucher type=1 对应之前的一码一验，凭证码数量必须与订单里的票数对应，核销时按照回传的凭证码进行核销；其他情况需回传 voucher type=0，此种情况下退款请求不会传凭证码给商家，核销时商家也可以不回传凭证码（按 used quantity 进行核销），需要码入园的产品商家照常返回凭证码即可，凭证码会照常展示。 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ vouchers | array<string> | 否 | 出票成功时为必传字段；凭证码数组，每个凭证码的最大长度为 varchar(20)；凭证码不区分大小写，只有大小写不同的两个码将会判定为重复码，导致出票失败。 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherPics | array<string> | 否 | 图片凭证码数组，出票成功时为必传字段；传图片链接地址, 有多个时, 顺序必须与 vouchers 的顺序一致；每个地址指向一张可下载的图片 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucherAdditionalList | array<VoucherAdditionalListSub> | 否 | 凭证附加信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucher | string | 是 | 必须存在于 `MtpOrderPayResponseBody` 中的 `vouchers` 中 `voucher` |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ additionMap | object<Map> | 是 | 凭证附加信息，key 值为“1”，表示为手机尾号后四位，value 为手机尾号后四位的值 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ realNameType | int | 否 | 实名制订单标识，取值：0、1； 0 代表非实名制订单(退款和核销无需传身份证信息)；1 代表实名制且需要凭证码订单(退款和核销需要凭码+身份证信息，即需要券码和游客信息相匹配)；不传默认为 0 非实名制订单 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialList | array<CredentialListSub> | 否 | 实名制证件、凭证码信息列表 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialType | int | 是 | 证件类型，0 表示身份证，1 表示护照，2 表示军官证，3 表示台胞证，4 表示港澳通行证，5 表示大陆居民往来台湾通行证，6 表示港澳居民来往内地通行证，7 表示外国人永久居留证，8 表示港澳台居民居住证 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ credentialNo | string | 是 | 证件号 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ voucher | string | 是 | 证件对应的凭证码，`realNameType` 为 1 时必传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionalList | array<OrderAdditionalListSub> | 否 | 订单附加信息列表 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionKey | int | 是 | 订单附加信息 Key，1 代表座位号 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderAdditionValue | string | 是 | 订单附加信息 Value，如果 `orderAdditionKey` = 1，则代表座位号的具体信息。如果有多个座位号，需要全部放到一起。`OrderAdditionValue` 字段推荐格式为 用户信息：座位号（姓名&凭证码&脱敏证件号&手机尾号：座位号信息），换行符使用 `\r\n`。 示例：游客:张三 座位号：B2-7；\r\n游客:李四 座位号：B2-8；\r\n游客:王二 座位号：B2-9；\r\n |
