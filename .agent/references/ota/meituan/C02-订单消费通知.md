### 订单消费通知

**更新时间：** 2026-03-09

**无需业务授权**

`meituan.trip.order.consume.notice` (订单消费通知)

#### 使用场景

当用户在合作方端进行消费后，合作方可主动调用此接口通知美团

#### 接口说明

1.  支持“全部消费”或“部分消费”的情况，合作方可通过接口中定义的“订单门票总数量”、“订单已使用门票总数量”以及“订单已退款门票总数量”进行调整，已完成核销。
2.  需要码核销的订单必须传入对应需要核销的门票凭证码信息（`voucherList`）, 以确保核销及时性与准确性；需要码核销按照订单的凭证核销，`voucherList`中的`voucher`、`status`正确。
3.  请合作方及时处理美团返回的消费失败的响应，可再推送消费通知。
4.  退款结果不能通过该接口返回。
5.  对于合作方无法对接该接口进行核销推送的情况，美团可以通过该查询接口主动查询订单与凭证码的状态进行核销。
6.  商家推送核销状态的接口，请真实核销后调用。若是虚假核销、提前核销，美团发现后会惩处商家。

#### 响应参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | String | 是 | 响应状态码，code 为 OP_SUCCESS 表示响应成功，其他状态码表示响应失败。具体系统错误码可以参考系统错误码。 |
| msg | String | 是 | 错误消息 |
| traceId | String | 是 | 请求消息唯一标识。 |
| data | Object | 是 | 业务数据，为 Json 对象，如果请求错误，该字段可能会展示错误的详细信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | int | 是 | 交互码，必须回传；200 为成功，300 为失败，201 为订单下所有票数已被核销或退款，202 为需要码核销传入凭证码列表中存在已核销/已退款的凭证码, 进行部分核销，701 为传入参数有错误，705 为凭证码异常订单无法核销，706 为需要码核销传入凭证码列表中与美团存储的未使用的凭证不一致，799 为其他 |

#### 业务错误码

| 错误码 | 错误描述 | 解决方案 |
| :--- | :--- | :--- |
| 200 | 成功 |  |
| 300 | 失败 | 请返回详细错误信息 |
| 201 | 订单下所有票数已被核销或退款 |  |
| 202 | 需要码核销传入凭证码列表中存在已核销/已退款的凭证码, 进行部分核销 |  |
| 701 | 传入参数有错误 | 检查传入参数 |
| 705 | 凭证码异常订单无法核销 | 请使用凭证码回传接口修复凭证码或联系美团产品 |
| 706 | 需要码核销传入凭证码列表中与美团存储的未使用的凭证不一致 | 检查传入参数 |
| 799 | 其他 | 根据返回详细信息处理 |
