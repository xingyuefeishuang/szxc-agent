### 订单关闭消息

**更新时间：** 2026-03-18

**无需业务授权**

---

#### 消息场景说明

`meituan.trip.order.close` (订单关闭消息)
1.  美团用户在下单后一定时间内未支付订单或主动取消订单，美团会主动将该订单关闭, 并通知商家该订单已关闭
2.  合作方下单/出票接口异常，例如合作方接口超时/合作方接口数据错误等，美团会主动将该订单关闭或退款, 并通知商家该订单已关闭
3.  商家可以选择对接此接口，进行必要操作，比如将该订单关闭或退款，以及占用账户资金进行释放。

---

#### 公共参数

*   **HTTP URL：** `https://baseUrl`（供应商提供）（仅支持 https 协议且端口为 443）
*   **HTTP Param：** param 遵循 JSON 格式，ContentType 为 `application/x-www-form-urlencoded`, charset=utf-8
*   **HTTP 方法：** POST
*   **消息推送公共参数：** 消息推送公共参数
*   **消息响应公共规范：** 消息响应公共规范

---

#### 美团请求参数

| 参数 | 类型 | 是否必传 | 描述 |
| :--- | :--- | :--- | :--- |
| msgType | Integer | 是 | 接口类型 ID：6610023 |
| developerId | Integer | 是 | 开发者 ID，在美团开发者网页直连申请时生成 |
| opBizCode | String | 否 | 业务实体 id |
| businessId | Integer | 是 | 产线 ID，默认传 66 |
| msgId | String | 是 | 本条请求的唯一标识，可用于去重判断 |
| timestamp | Long | 是 | 消息推送时间戳 |
| sign | String | 是 | 消息的安全签名，可以根据此签名判断消息真伪。开发者需要对请求消息的签名进行验证，签名规则见：签名规则 |
| message | String | 是 | json 格式的消息数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | 是 | 合作方 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | object<MtpPriceRequestBody> | 是 | 美团订单 id |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ orderId | long | 是 | 合作方产品 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ closeType | int | 是 | 关闭类型 1: 用户未支付，美团关闭订单，请合作方接收此通知后同步关闭订单 2: 合作方下单接口异常，美团关闭订单，请合作方接收此通知后同步关闭订单 3: 合作方出票接口异常，美团出票失败且已退款，请合作方接收此通知后根据实际订单状态关闭订单或退款 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | String | 否 | 合作方订单 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ refundSerialNo | String | 否 | 美团退款流水号 |

---

#### 请求示例

> **注意：** 回调消息统一通过 HTTPS 请求推送，Post 方法，ContentType 为 `application/x-www-form-urlencoded`。开发者需要按照规范实现 HTTPS 接口，以接收推送的消息。

**请求示例**
```json
{
    "msgType": "6610023",
    "opBizCode": "1711591987",
    "developerId": "123456",
    "businessId": "66",
    "sign": "4656285a4c2493e279d929b8b9f4e29310da8b3b",
    "msgId": "3702923312382104528",
    "timestamp": "1711592316026",
    "message": "{\"partnerId\":1,\"body\":{\"orderId\":1000,\"closeType\":1,\"partnerOrderId\":\"example\",\"refundSerialNo\":\"example\"}}"
}
```

---

#### 供应商返回参数

结果数据应为 json 格式，放在 http body 中返回。

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | Integer | 是 | 0 为正常，非 0 为异常 |
| message | String | 否 | 错误信息 or "success"，出错情况下，存放出错信息，用于问题定位。 |
| data | String | 否 | 在收到消息回调后若需主动返回数据，使用此字段 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | Integer (int32) | 是 | 交互码 取值请参考本接口的交互码类型映射表 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | 是 | 交互描述 字符串 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | Integer (int32) | 是 | 合作方 ID |

---

#### 返回示例

```json
{
    "code": 0,
    "message": "success",
    "data": "{\"code\":200,\"describe\":\"success\",\"partnerId\":36609}"
}
```
