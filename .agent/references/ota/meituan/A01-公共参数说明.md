# 公共参数说明

**更新时间**：2025-11-04

## API公共参数
适用场景：三方开发者调用美团合作中心服务接

### 请求协议说明
技术合作中心提供的业务API，目前只支持HTTP POST一种请求方式。

指定Body Content-Type为：application/x-www-form-urlencoded

Post请求时，所有参数均通过表单传递，请勿将请求参数放到Query参数中。

### 参数说明
接口参数分为两部分：系统参数和业务参数。

#### 系统参数
请求所有接口均需要携带的参数，用于开发者身份认证和数据安全校验。系统参数包括：

| 参数 | 类型 | 名称 | 示例值 | 是否必须 |
| :--- | :--- | :--- | :--- | :--- |
| developerId | int | 开发者id，开发者中心申请的开发者ID | 100567 | 必须 |
| charset | string | 交互数据的编码【utf-8】 | utf-8 | 必须 |
| timestamp | int | 当前请求的时间戳【单位是秒】 | 1618975600 | 必须 |
| version | int | 接口版本，当前版本为【2】,目前只能传2，不能不传！ | 2 | 必须 |
| businessId | int | 业务类型id 见本文下方【业务ID列表】<br>如团购:1 外卖:2 | 66 | 必须 |
| sign | string | 请求的数字签名，是通过所有请求参数通过摘要生成的，保证请求参数没有被篡改。<br>签名规则 | ce5141614230cd561dfed062857544f226b1cd54 | 必须 |
| biz | jsonString | 业务请求参数，格式为jsonString，详细说明见下方【业务参数】。<br>该参数为必填参数，如果接口不需要业务参数，请传空json对象字符串 "{}" | {"id":1,"name":"hello"} | 必须 |
| appAuthToken | string | 授权获取的token（通过门店映射或三方授权获取）<br>授权接入 | V2-0ecde8631431a5ed6b3e7368afbabdabf5c3ff6e5bed446df0b7c99e7a6df054818c322721429afad0078d6d | 非必须 |

#### 业务参数
业务参数是指调用具体某个接口时需要的和业务相关的参数，通常不同的接口有不同的业务参数。

业务参数统一使用JSON字符串表示，并将业务此JSON字符串作为指定参数"biz"的值。

假如一个接口需要两个参数：id和name，传参的方式可参考：

**代码块**
**Java**
```java
appAuthToken:V2-5fc970ded1f0f933594ee68bf8da6eaa06a44a340e5b23832f944a716379f006aeddd4598541271ece7f91d185b64080 
charset:UTF-8 
timestamp:1618975600 
version:2 
developerId:1031123 
businessId:18 
sign:96e2cedadeb7401acf48e46672673f6512cc460e 
biz:{"id":1,"name":"hello"} 

---

### ⚠️ 注意事项

> **对于 POST 请求，请求参数全部放在 Body 里，不要把参数放在 Query 里。**

### 响应格式

所有接口响应 `Content-Type` 均为：`application/json`，即响应均为 JSON 格式。接口响应有统一的格式：

#### 成功格式

```json
{
    "code": "OP_SUCCESS",
    "data": {"partnerId":26465, "code":200},
    "traceId": 5362971231773781183
}
```

-   **code**: 为 `OP_SUCCESS` 表示请求成功，其他情况都为请求失败。
-   **data**: 接口请求成功时，此字段表示接口响应的业务数据。

#### 失败格式

```json
{
    "code": "OP_SYSTEM_PARAM_ERROR",
    "msg": "缺少系统参数",
    "traceId": 5362971231773781183
}
```

-   **code**: 定义参见错误码定义。
-   **msg**: 错误描述信息。
-   **traceId**: 本次调用产生的流水号，接口调用异常，可以提供此流水号用于排查问题。

### 消息公共参数

**适用场景：** 开发者可通过对接消息回调，接收美团主动推送的消息通知或业务数据，如新订单生成、门店信息变更通知。

#### 消息回调接收方式

**消息格式**

回调消息统一通过 HTTP 请求推送，Post 方法，`ContentType` 为 `application/x-www-form-urlencoded`。开发者需要按照规范实现 HTTP 接口，以接收推送的消息。

-   对于“业务回调”类型的消息，消息推送的具体数据格式请参考具体的回调接口文档。
-   对于“统一回调”类型的消息，消息推送的数据格式如下：

| 参数 | 类型 | 是否必须 | 说明 |
| :--- | :--- | :--- | :--- |
| msgType | int | 是 | 消息类型 |
| timestamp | long | 是 | 消息推送时间戳（秒） |
| sign | string | 是 | 消息的安全签名，可以根据此签名判断消息真伪。开发者需要对请求消息的签名进行验证，签名规则见：签名规则 |
| developerId | int | 是 | 开发者 id |
| businessId | int | 是 | 消息所属的业务 id |
| msgId | string | 是 | 本条消息的唯一标识，可用于去重判断 |
| opBizCode | string | 否 | 业务实体 id |
| ePoiId | string | 否 | 服务商门店 id（外卖/团购等业务适用） |
| message | string | 是 | json 格式的消息数据 |

#### 回调地址要求

-   回调地址需要为 HTTPS 域名地址，不支持配置 HTTP 接口格式。
-   建议使用正规供应商提供的域名，避免直接使用 IP 地址。

#### 回调地址如何配置

开发者可以为不同的消息回调类型分别设置用于接收回调的 HTTPS URL 地址，设置入口在开发者后台 -> 我的开发者 -> 回调接口设置。新增或修改回调地址后，需要联系运营审核通过后方可生效。

![回调配置](image.jpeg)

#### 消息订阅

对于“统一回调”类型的消息，可以通过订阅开关控制是否需要接收消息数据，配置入口在开发者后台 -> 我的开发者 -> 消息订阅。

> 如果在此处无法看到想要订阅的消息，请先联系运营开通消息订阅权限。

#### 返回接收结果

开发者在收到消息回调数据后，应在规定秒数内返回结果，具体可查看消息超时配置。若超时未返回，技术合作中心会将本次消息回调的请求作为失败处理，影响开发者 SLA。

开发者在处理消息回调后，应按照规范返回处理结果。结果数据应为 JSON 格式，放在 http body 中返回。

| 字段 | 类型 | 名称 | 是否必须 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| code | int | 状态码 | 是 | **0 为正常，非 0 为异常**。此 code 码代表交互状态，非 0 代表交互失败，业务 code 码在 data 字段中传递 |
| message | string | 消息字段 | 否 | 错误信息 or "success"，出错情况下，存放出错信息，用于问题定位。 |
| data | string | 业务数据 | 是 | 在收到消息回调后返回业务数据，使用此字段 |

**返回示例：**

**成功返回**
```json
{
    "code": 0,
    "message": "success",
    "data": "{\"code\": 200, \"partnerId\": 26465, \"body\":{\"orderId\":1234567890}}"
}
```

**交互失败返回**
```json
{
    "code": -99,
    "message": "网络错误！"
}
```

**业务失败返回**
```json
{
    "code": 0,
    "message": "success",
    "data": "{\"code\":300,\"describe\":\"库存不足！\"}"
}
```
