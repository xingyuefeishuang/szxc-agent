### 🔐 鉴权说明

**更新时间：** 2025-11-04

为了保证美团技术服务合作中心与服务商之间数据交换的完整性和真实性，美团技术服务合作中心在服务商调用接口时做签名校验：

美团技术服务合作中心采用 SHA1 摘要算法计算签名，验签逻辑为：
**请求发起方（计算签名，请在请求中带上签名） ---> 请求接收方（按请求参数计算签名，并校验签名）**

签名不一致的请求应当被拒绝，所以双方需要使用相同签名计算方法，保证最终签名的一致。

*   **请求方式：** HTTP POST
*   **Content-Type：** application/x-www-form-urlencoded
*   **参数位置：** Post 请求时，所有参数均通过表单传递，请勿将请求参数放到 Query 参数中。

#### 请求参数列表

| 参数 | 类型 | 最大长度 | 是否必填 | 描述 | 示例值 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| sign | String | - | 是 | 数字签名 | ceb760af62431e20fd2e4aa1bbf21d2c17012dc0 |
| timestamp | Long | - | 是 | 时间戳，单位秒 | 1577771730 |
| charset | String | - | 是 | 字符编码 | utf-8 |
| developerId | Long | - | 是 | 服务商身份标识 | 123123 |
| version | String | - | 是 | 版本号，固定传 2 | 2 |
| businessId | int | - | 是 | 业务类型 id，门票业务固定传 66 | 66 |
| biz | String | - | 是 | 业务请求参数 | {"ticket":"10346762252","number":2} |
| appAuthToken | String | - | 否 | 门店令牌，接口需要授权时必填 | abc |

> **签名规则说明文档：** [点击查看](https://developer.meituan.com/docs/biz/comm-dev-isv-sign-rule)

#### 🧮 签名计算方式

以执行核销通知接口 `/rhone/mtp/api/order/consume/notice` 为例，假如开发者的 `signKey` 是 `k4ke1u0r44bgq1kx`，请求的参数如下：

*   charset：UTF-8
*   timestamp：1761737557
*   developerId：117092
*   biz: `{"partnerId":26465,"body":{"orderId":5008017280866233593,"partnerOrderId":"partnerOrderId-5008017280866233593","quantity":1,"usedQuantity":1,"refundedQuantity":0,"voucherList":[{"voucher":"voucher-5008017280866233593","voucherPics":"www.meituan.com","voucherInvalidTime":"2025-10-29 19:00:00","quantity":1,"status":1}]}}`
*   version: 2
*   businessId: 66

**计算步骤：**

1.  **参数排序与拼接**
    将请求参数中除 `sign` 外的多个键值对，根据键按照字典序排序，并按照 "key1value1key2value2..." 的格式拼成一个字符串。
    **拼接结果：**
    `biz{"partnerId":26465,"body":{"orderId":5008017280866233593,"partnerOrderId":"partnerOrderId-5008017280866233593","quantity":1,"usedQuantity":1,"refundedQuantity":0,"voucherList":[{"voucher":"voucher-5008017280866233593","voucherPics":"www.baidu.com","voucherInvalidTime":"2025-10-29 19:00:00","quantity":1,"status":1}]}}businessId66charsetutf-8developerId 117092timestamp 1761737557version2`

2.  **构造待签名字符串**
    将 `signKey` 拼接在步骤 1 中排序后的字符串前面。
    **待签名字符串：**
    `k4ke1u0r44bgq1kxbiz{"partnerId":26465,"body":{"orderId":5008017280866233593,"partnerOrderId":"partnerOrderId-5008017280866233593","quantity":1,"usedQuantity":1,"refundedQuantity":0,"voucherList":[{"voucher":"voucher-5008017280866233593","voucherPics":"www.baidu.com","voucherInvalidTime":"2025-10-29 19:00:00","quantity":1,"status":1}]}}businessId66charsetutf-8developerId 117092timestamp 1761737557version2`

3.  **SHA1 加密**
    使用 sha1 算法加密待加密字符串并转为小写即为 `sign`。
    **生成签名：**
    `37350dbf19b73248c9d440ea13cad18b13ccada7`

4.  **发起请求**
    将 `sign` 添加到请求参数中。

**HTTP 请求示例 (cURL)：**

```json
curl --location 'https://api-open-cater.meituan.com/rhone/mtp/api/order/consume/notice' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'sign=3dc942507dd30c6e49695e88067339ae55cd715d' \
--data-urlencode 'developerId=117092' \
--data-urlencode 'charset=utf-8' \
--data-urlencode 'timestamp=1761737557' \
--data-urlencode 'version=2' \
--data-urlencode 'businessId=66' \
--data-urlencode 'biz={"partnerId":26465,"body":{"orderId":5008017280866233593,"partnerOrderId":"partnerOrderId-5008017280866233593","quantity":1,"usedQuantity":1,"refundedQuantity":0,"voucherList":[{"voucher":"voucher-5008017280866233593","voucherPics":"www.baidu.com","voucherInvalidTime":"2025-10-29 19:00:00","quantity":1,"status":1}]}}'

关于 appAuthToken 字段：
接口需要授权时必填，直连门票业务所有接口无需授权，故该字段无需填写。
