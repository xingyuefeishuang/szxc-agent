### 多层价格日历变化通知 V2

**更新时间：** 2026-03-09

**无需业务授权**

`meituan.trip.product.level.price.notice.v2` (多层价格日历变化通知 V2)
1.  当美团异步（`asyncType=1`）拉取价格日历时，供应商通过这个接口推送美团场次层级信息及价格库存信息。
2.  供应商价格日历或场次信息变化，可以通过这个接口推送给美团。

---

#### 请求地址

| 环境 | HTTPS 请求地址 |
| :--- | :--- |
| 正式环境 | `https://api-open-cater.meituan.com/rhone/mtp/api/level/price/notice/v2` |

---

公共请求参数

#### 请求参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| charset | String | 是 | 字符编码 |
| developerId | Long | 是 | 服务商身份标识 |
| sign | String | 是 | 数字签名 |
| businessId | int | 是 | 业务类型 id，固定传 66 |
| version | String | 是 | 版本号，固定传 2 |
| timestamp | Long | 是 | 时间戳 |
| biz | jsonString | 是 |  |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerDealId | String | 是 | 合作方产品 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ startTime | String | 是 | 价格日历的开始时间，格式：yyyy-MM-dd |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int | 是 | 合作方 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ endTime | String | 是 | 价格日历的结束时间，格式：yyyy-MM-dd |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | array<BodySub> | 是 | 层级及价格库存信息数据 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ marketPrice | double | 是 | 市场价，单位：元，保留两位小数 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ priceDate | String | 是 | 价格日历的日期, 格式 yyyy-MM-dd eg：2022-10-01 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ mtPrice | double | 是 | 美团售卖价，单位：元，保留两位小数 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ settlementPrice | double | 是 | 结算价，单位：元，保留两位小数 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerPrimaryKey | String | 是 | sku 维度的唯一标识 id，建议生成方式：MD5(skuInfo.toJsonString+priceDate) |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ stock | int | 是 | 日库存(指剩余可售卖库存) |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ attr | String | 否 | 扩展数据，sku 信息，如：`"{\"playtime\":\"09:00-10:00\",\"ticketcode\":\"贵宾席\",\"stock\":100}"` |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ skuInfo | object<SkuInfo> | 是 | sku 维度信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ startTime | String | 是 | 场次开始时间，要求格式：HH:mm，eg：15:30 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ endTime | String | 是 | 场次结束时间，要求格式：HH:mm，eg：20:59 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ levelInfoList | array<LevelInfoListSub> | 是 | 场次信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ levelName | String | 是 | 层级名称信息，如：”上午场“、”贵宾席“ |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ levelNo | int | 是 | 层级序号，最多支持 3 层：1-第一层；2-第二层；3-第三层 |

#### 响应参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | String | 是 | 响应状态码，code 为 OP_SUCCESS 表示响应成功，其他状态码表示响应失败。具体系统错误码可以参考系统错误码。 |
| msg | String | 是 | 错误消息 |
| traceId | String | 是 | 请求消息唯一标识。 |
| data | Object | 是 | 业务数据，为 Json 对象，如果请求错误，该字段可能会展示错误的详细信息 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | int |  | 交互码，200 为成功，300 为失败 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String |  | 响应描述 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | int |  | 合作方 ID |
