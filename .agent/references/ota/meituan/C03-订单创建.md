### 订单创建

**更新时间：** 2026-03-19

**无需业务授权**

---

#### 消息场景说明

`meituan.trip.order.create` (订单创建 V2)

##### 使用场景

美团调用此接口请求合作方创建订单（预约，但未支付）

##### 接口说明

1.  下单失败的 `describe` 要返回真实的失败原因，不能用“其他”代替。`code` 按照美团给出的对接。
2.  创建订单过程中我们提供该次订单的结算价, 如果需要校验结算价请在本接口中进行。
3.  此接口为必须接口，若无该接口会重新对接。
4.  该接口通用超时底线时间：连接 5s 读取 15s，可调整但不能总和不能超过 20s。

---

#### 公共参数

*   **HTTP URL：** `https://baseUrl`（供应商提供）（仅支持 https 协议且端口为 443）
*   **HTTP Param：** param 遵循 JSON 格式，ContentType 为 `application/x-www-form-urlencoded`, charset=utf-8
*   **HTTP 方法：** POST
*   **消息推送公共参数：** 消息推送公共参数
*   **消息响应公共规范：** 消息响应公共规范

#### 供应商返回参数

| 名称 | 类型 | 是否必须 | 描述 |
| :--- | :--- | :--- | :--- |
| code | Integer | 是 | 0 为正常，非 0 为异常 |
| message | String | 否 | 错误信息 or "success"，出错情况下，存放出错信息，用于问题定位。 |
| data | String | 否 | 在收到消息回调后若需主动返回数据，使用此字段 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ code | Integer | 是 | 交互码，必须回传；其中 200 为订单创建成功，401 为虚拟账户余额不足，402 为账户过期，403 为预付款用尽，410 为规则校验失败，420 为没有库存，421 为产品已经下线，422 为当前日期没有价格日历，499 为其他 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ describe | String | 是 | 交互描述，如错误信息等。失败时要返回真实的失败原因，不能用“其他”代替，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerId | Integer | 是 | 合作方 ID，必须回传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ body | array<MtpOrderCreateResponseBody> | 否 | 创建订单返回内容，下单成功时必传 |
| &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└ partnerOrderId | String | 是 | 合作方订单 id，下单成功必须回传 |
